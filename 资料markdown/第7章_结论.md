# 第7章  结论

在开发草地病虫害监测系统的过程中，我贴合用户需求，完成了系统核心功能的设计及实现，包含用户登录认证、草地档案管理、田间场地管理、种植培育跟踪、病虫害防治与预警、物资仓储管理、收获销售管理、资产统计分析以及大模型问答助手等功能模块。在整个开发过程中，通过绘制流程图、用例图以及编写用例规约等方式，对系统的各项需求进行了全面的分析，然后重点设计并实现了系统的核心功能模块，并通过持续的测试与优化，使系统的功能逐步完善，基本满足了初期的应用目标和使用需求。

通过草地档案管理模块，用户可以实现作物基础信息的增删改查和批量导入，按品种、状态等条件快速检索档案；病虫害防治与预警模块实现了施药防治记录的登记和自动预警提醒，帮助管理人员及时掌握防护周期；物资仓储模块实现了农资商品的入库、出库和库存查询，余量不足时自动高亮提示；收获销售模块支持销售订单登记和资产估值自动扣减；资产统计模块通过数据可视化图表展示资产分布和收入趋势，为经营决策提供数据支撑。整个系统各模块的设计都围绕用户的实际需求和使用感受展开，以满足不同管理人员获取信息和执行操作的要求。

系统根据用户身份的不同，设置了不同的权限范围，实现了功能权限的差异化管理。登录后的用户可浏览和管理草地档案、登记田间记录、查询预警信息、管理物资出入库、登记销售订单以及查看资产统计报表；管理员拥有更高的权限，能够管理监测站点和地块信息、维护农资商品资料、配置预警阈值以及查看系统的各类统计数据。通过这种角色区分，不仅满足了种植管理人员日常业务操作的需求，也为后台管理人员提供了高效便捷的管理工具。

本系统在设计上还存在一些不足之处，例如前端页面的展示效果还有待提升，整体界面风格不够精致，部分查询响应速度在数据量较大时偏慢，大模型问答的准确性在复杂问题上仍需提升，移动端适配也尚未完成。未来在保持功能稳定的前提下，可以逐步引入Redis缓存优化查询性能、优化大模型的系统提示词提升问答质量、开发移动端页面支持现场操作，使系统不仅实用，而且更加高效和便捷，以此更好地服务于草地作物种植管理工作。
