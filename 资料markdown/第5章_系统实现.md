# 第5章  系统实现

## 5.1  开发环境

本系统采用前后端分离的设计思路和浏览器/服务器（B/S）架构形式，后端进行开发时采用的环境为VS Code，基于Python Flask框架进行开发，依托它轻量灵活和快速开发的能力，把复杂业务逻辑的实现过程简化了。前端部分开发的环境选定为VS Code，依靠Vue.js 3框架搭建，充分发挥它组件化与响应式的特性，为用户呈现了一个直观且互动效果好的操作界面。结合像Element Plus这样的组件库，可快速搭建出美观且功能丰富的页面。采用Navicat Premium来管理数据库，选择MySQL作为核心数据库管理系统，因其具备高安全性、数据一致性和访问效率高等优点。同时，通过集成智谱GLM大模型API，给出了农业知识问答与智能数据分析，让实用性得到进一步增强。

**表5.1  开发环境配置**

| 环境项 | 配置参数 |
|--------|---------|
| 操作系统 | Ubuntu 22.04 LTS / Windows 11 |
| 开发工具 | VS Code 1.85、Navicat Premium 16 |
| 后端语言 | Python 3.9 |
| 后端框架 | Flask 3.0.3、Flask-SQLAlchemy 3.1.1、Flask-JWT-Extended 4.6.0 |
| 前端框架 | Vue.js 3.4.31、Element Plus 2.9.8、Vite 5.3.2 |
| 数据库 | MySQL 5.7 |

| 浏览器 | Google Chrome 120+ / Mozilla Firefox 121+ |

系统采用Docker Compose进行统一部署，生产环境通过三个容器分别运行MySQL数据库、Flask后端服务和Nginx前端代理。开发人员本地调试时，后端直接使用Flask内置服务器运行，前端通过Vite开发服务器提供热更新支持。

## 5.3  本章小结

本章介绍了草地病虫害监测系统的开发和部署环境，接着详细介绍了核心功能模块是如何设计与实现的。主要实现了用户登录认证、草地档案与田间管理、种植培育、病虫害防治与预警、物资仓储与收获销售、资产统计与分析以及大模型问答助手等功能模块。用户登录认证模块通过Flask-JWT-Extended实现了基于JWT的无状态身份验证和令牌刷新机制；草地档案模块实现了多条件组合查询和Excel批量导入；病虫害防治模块通过跨表关联查询补全了列表展示信息，并利用APScheduler定时任务每日自动生成预警提醒；物资仓储模块在出库时执行库存校验和事务扣减，保证了数据一致性；资产统计模块通过批量更新和插入操作提升了大数据量下的处理效率；大模型问答助手通过Function Calling实现了模型与数据库的联动查询，并以SSE流式传输提升了交互体验。各模块的实现均基于Flask蓝图和Vue组件化的技术架构，通过统一的数据格式和错误处理机制保证了前后端交互的稳定性。
