# 第3章  需求分析

## 3.1  系统目标及流程分析

### 3.1.1  系统目标

本系统的核心目标是为草地作物种植管理提供一个统一的信息化操作平台，将原先分散在纸质档案和电子表格中的业务数据迁移到线上集中管理。具体包括以下几个方面的目标：一是建立完整的草地作物基础档案，记录每块监测地块的作物品种、播种日期、生长状态和受灾等级等信息，支持按条件检索和批量导入导出；二是实现田间场地和种植培育过程的数字化跟踪，包括监测站点的维护消毒记录、作物的生长周期管理和采样调查数据；三是构建病虫害防治与预警机制，记录施药防治和防护措施的详细信息，通过定时任务自动计算防护到期时间并生成预警待办；四是管理物资仓储和收获销售业务，跟踪农药肥料的出入库和库存变化，记录收获后的销售结算和经济效益；五是提供资产统计和数据分析功能，自动计算每日资产估值和收入情况，生成可视化图表辅助经营决策；六是通过大模型问答助手为管理人员提供农业知识查询服务，辅助解决日常生产中的技术问题。

### 3.1.2  系统主要流程分析

系统的主要业务流程可以从日常管理的实际工作环节来理解。管理人员首先需要登录系统，通过用户名和密码验证身份后进入主界面。在日常工作中，管理人员会定期录入或更新草地作物的基础档案信息，包括新增种植批次、修改生长状态、记录田间长势调查数据等。当田间出现病虫害情况时，管理人员在系统中登记施药防治记录，系统根据药剂的使用周期自动计算下次防治的到期时间，并在预警模块中显示待办提醒。物资入库时，管理人员录入农药或肥料的采购信息，出库时记录使用数量和用途，系统自动更新库存余量。收获季节，管理人员登记销售订单，系统根据销售记录自动扣减对应批次的资产估值并生成日收入报表。管理人员还可以随时查看资产统计图表，了解当前在田作物的总体价值和各品种占比情况。整个流程中，大模型问答助手作为辅助工具，可以随时回答管理人员关于病虫害识别、防治方案选择等方面的咨询。

#### （1）用例图

系统用户的用例图主要包括登录认证、草地档案维护、田间长势记录、监测站点管理、监测地块管理、种植培育跟踪、生长周期管理、采样调查记录、施药防治登记、防护措施记录、病虫害预警查看、农资商品维护、物资入库管理、物资出库管理、库存盘点查询、收获销售登记、销售结算统计、资产估值计算、资产标准维护、日收入报表查看、数据可视化展示、农业知识问答等功能。用户在系统中通过登录验证后，可以在草地档案模块新增、编辑、删除和查询作物基础档案，按作物编号、品种、状态等条件进行多维度检索，并支持Excel模板批量导入历史数据；在田间管理模块维护监测站点和地块信息，记录定期田间调查和长势监测数据；在种植培育模块跟踪作物从萌芽、拔节、开花到成熟的完整生长历程，登记采样调查和炼苗记录；在病虫害防治模块登记施药防治和防护措施详情，系统自动根据药剂使用周期和阈值设置计算到期时间，并在预警模块生成待办提醒；在物资仓储模块管理农药、肥料等农资商品的采购入库和使用出库，实时查看库存余量和低库存预警；在收获销售模块登记销售订单，系统自动扣减资产估值并生成日收入报表；在资产统计模块查看按品种分组的资产估值明细和占比图表，按日期回溯历史资产变化；还可以通过大模型问答助手咨询病虫害识别、防治方案选择等农业技术问题。如图3.1所示。

### 3.2.2  非功能需求分析

除了功能需求外，系统还需要满足以下几方面的非功能需求：

**性能需求**：在正常网络环境下，前端页面首次加载时间不超过3秒，常规数据查询接口的响应时间不超过500毫秒。系统支持至少20个用户同时在线操作，数据库查询支持分页加载，单次返回数据量不超过1000条记录，避免一次性加载大量数据导致前端卡顿。

**可靠性需求**：系统需要保证数据的完整性和一致性，涉及多表更新的业务操作（如销售扣减资产）必须使用数据库事务，确保要么全部成功要么全部回滚。后端定时任务（如每日预警更新）需要设置异常捕获和日志记录，任务失败时能够记录错误信息但不影响主服务的正常运行。

**易用性需求**：前端界面应简洁直观，表格操作支持搜索、筛选、排序和分页，表单填写提供明确的必填项提示和数据格式校验。常用功能（如新增记录）的入口应明显易找，减少用户的操作步骤和学习成本。系统支持Excel模板批量导入基础档案数据，避免逐条录入的低效工作。

**可维护性需求**：后端代码采用蓝图模块化组织，各业务模块的路由、视图和模型相对独立，便于后续功能扩展和问题定位。数据库表结构变更应通过Alembic迁移脚本管理，避免直接修改生产环境数据库。系统采用Docker容器化部署，配置信息通过环境变量注入，便于在不同环境（开发、测试、生产）之间快速迁移。

**安全性需求**：用户密码采用加密存储，禁止明文保存。前后端通信通过JWT令牌进行身份验证，敏感接口（如数据修改）必须校验令牌有效性。系统对输入数据进行清洗和校验，防止SQL注入和跨站脚本攻击。生产环境禁止输出调试日志，避免泄露数据库连接信息和API密钥。

## 3.3  本章小结

本章从系统目标和业务流程出发，对草地病虫害监测系统的需求进行了全面分析。首先明确了系统的六大核心目标，梳理了从登录、档案维护、田间管理、病虫害防治、物资仓储、收获销售到资产统计的完整业务流程。随后采用用例分析方法，详细描述了七个功能模块共十二个核心用例的前置条件、基本流程和异常处理，涵盖了用户登录、档案维护、田间记录、生长跟踪、施药防治、预警查看、物资出入库、销售结算、资产估值、收入报表和知识问答等主要业务场景。最后从性能、可靠性、易用性、可维护性和安全性五个方面阐述了系统的非功能需求。上述需求分析为后续的系统总体设计和详细实现提供了明确的依据和边界。
