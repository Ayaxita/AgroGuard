## 一、操作概述

本次操作对 AgroGuard（草地作物病虫害智能监测系统）进行了全面的代码审查、Bug修复、前后端对齐、数据库验证和Docker重新部署。目标是消除系统中所有已知的致命错误和严重错误，使前后端数据库完全对应，代码规范统一，最终构建成一个零Bug、可流畅运行的系统。

### 2.1 项目基本信息
- **技术栈**：Python 3.9 + Flask + SQLAlchemy + Vue 3 + TypeScript + MySQL 5.7 + Docker
- **服务状态**：MySQL、Flask、Vue 三个容器正在运行，但运行的是旧版本代码

### 2.2 发现的主要问题

#### 🔴 P0 - 致命错误
1. `import datetime` 导致 `AttributeError`（8个views文件）
2. 模型字段名与数据库字段名严重不匹配（会导致SQL 1054错误）
3. 前端大量模块错误调用 `/basic/manuinfo`
4. 前端仍使用旧的 `/colony/` 前缀

#### 🟠 P1 - 严重错误
1. 密码重置未进行MD5哈希比较
2. 大量前端模块API路径缺少前导 `/`
3. `h_store/views.py` 拼写错误 `yaoping_fees`

#### 🟡 P2 - 中等问题
1. 路由重复定义（propagation/hardeninginfo/add）
2. 跨蓝图路由冲突（statistic中的/basic/basicinfo/update_grandparents）
3. import严重重复和混乱（statistic/view.py）

#### 🟢 P3 - 低优先级
1. 菜单从本地JSON加载（静态权限控制）
